C. 实验结论分布

  • 在 46GB 单卡 + offloading 条件下，no-SD 与 EAGLE-3 都已跑通 serve benchmark。
  • 当前 workload (input=256, output=64, qps=1) 下，no-SD 指标优于 EAGLE-3。

D. 模块贡献分布

  • offloading：使 30B MoE 在单卡上可加载并完成基准。
  • 本地模型路径：避免远程下载不稳定导致的启动失败。
  • eagle3 speculator：可与 target 同机加载，但会挤压 KV cache 空间。

E. 当前推荐配置

推荐先使用 no-SD 基线进行大规模 workload 扫描，再在子集 workload 上评估 EAGLE-3。
建议启动参数：
- MAX_MODEL_LEN=512
- CPU_OFFLOAD_GB=30
- SWAP_SPACE_GB=64
- EAGLE-3 额外使用 SPEC_TOKENS=2 并启用本地 speculator 目录

F. 当前主要问题

  1. EAGLE-3 在显存紧张时容易出现 KV cache 可用空间不足。
  2. 当前 bench 产物缺少 goodput 字段，需在后处理阶段单独补算。
  3. 需要继续扩展 workload profile（prompt/output/rate）验证结论稳健性。
