\title{\sysname: Effective Speculative Decoding for Offloaded Mixture-of-Experts Inference on a Single GPU}

\begin{abstract}
CPU offloading enables commodity-GPU serving of Mixture-of-Experts (MoE) models, but efficient decoding then critically depends on expert caching. Since speculative decoding (SD) delivers substantial speedups on dense models, it is a natural candidate for further accelerating cache-assisted offloaded MoE decoding. However, SD performs worse than expected: multi-token verification substantially expands the per-layer expert working set, yet even when the cache is large enough to hold SD's steady-state working set, existing AR-optimized cache designs fail to translate this into proportional speedup. We analyze the interaction between cache capacity and SD effectiveness, and derive a memory threshold $M^*$ that marks a phase transition: below $M^*$, SD provides no benefit regardless of system optimizations; above $M^*$, two practical barriers still prevent SD from reaching its full potential, namely overhead on the cache-hit path and transient working-set overflow that triggers cascading evictions. To address these barriers, we present \sysname, a system with two complementary designs: 1)~\emph{Unblock} restores an HBM-speed cache-hit path by enabling direct execution on cache-resident experts, reusing stable expert-to-slot mappings on the common path, and overlapping residual misses with computation; 2)~\emph{Protect} pre-stages likely verification experts during draft generation, preventing transient working-set bursts from cascading into sustained cache misses. Extensive experiments on Qwen3-30B-A3B with EAGLE-3 show that \sysname achieves up to $4.80\times$ speedup over the offload baseline, increasing decode throughput from 2.08 to 9.98~tok/s.
\end{abstract}
